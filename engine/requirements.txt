# Speed: cupy softsplat path (validated on RTX 5090 / sm_120). cupy-cuda13x uses cuda-pathfinder to
# locate the CUDA 13 runtime, so the old nvrtc DLL-search shim is no longer load-bearing; the nvrtc +

# unsuffixed names below (they resolve to 13.x).

# TensorRT for RTX backend (default; --no-trt opts out): native engines for the 5 GMFSS sub nets, the
# RIFE IFNet and the --restore pass, about 2.2x over the cupy fp16 path. NOT pip-installable on this
# runtime: NVIDIA ships tensorrt_rtx wheels only up to cp313 and the runtime is Python 3.14, so the
# module is our own pybind11 build - see engine/trtrtx_bindings/BUILD.md (one build.cmd run installs
# tensorrt_rtx.pyd + its DLLs into site-packages). When NVIDIA ships a cp314 wheel, swap back to
# `tensorrt-rtx-cu13`. If the module is absent, trt_runtime.py drops to the eager path. onnx +
# onnxscript drive the dynamo ONNX exporter used to build the engines.
onnx
onnxscript
