# Speed: cupy softsplat path (validated on RTX 5090 / sm_120). cupy-cuda13x uses cuda-pathfinder to
# locate the CUDA 13 runtime, so the old nvrtc DLL-search shim is no longer load-bearing; the nvrtc +
# runtime wheels are still listed so they are present. NOTE: the CUDA 13 wheels DROPPED the -cuXX
# suffix - the nvidia-cuda-*-cu13 packages are deprecated placeholders that fail to build, so use the
# unsuffixed names below (they resolve to 13.x).
cupy-cuda13x
nvidia-cuda-nvrtc
nvidia-cuda-runtime

# TensorRT for RTX backend (default; --no-trt opts out): native engines for the 5 GMFSS sub nets, the
# RIFE IFNet and the --restore pass, about 2.2x over the cupy fp16 path. `tensorrt-rtx-cu13` is TensorRT
# for RTX (CUDA 13.3 libs, matching torch's cu130): compact (~0.2 GB of libs), hardware-agnostic AOT
# engines, near-instant builds; if it is absent, trt_runtime.py drops to the eager path. onnx +
# onnxscript drive the dynamo ONNX exporter used to build the engines.
tensorrt-rtx-cu13
onnx
onnxscript
